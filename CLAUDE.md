## What this repo is

A static personal resume site served via GitHub Pages at `https://ppezzull.github.io/`. There is no runtime application — the pipeline is: JSON data → themed HTML → deploy. The only build step is `src/generate-resume.js`.

## Commands

```bash
npm install --include=dev   # dev deps ARE required (React/Babel) — `--include=dev` matters; NODE_ENV=production or npm `production=true` skips them
npm run generate            # node src/generate-resume.js → writes src/resume.html
```

There are no tests. `npm test` is the npm default placeholder that exits 1.

To preview: open `src/resume.html` directly (the theme renders with `../fonts/` paths, which resolve from `src/`).

## Architecture

The build is a single Node script (`src/generate-resume.js`) that runs server-side rendering of a React-based theme and post-processes the resulting HTML with string ops + Cheerio:

1. **`setupBabel()`** — registers `@babel/register` so the ESM/JSX theme in `node_modules/jsonresume-theme-professional` can be `require()`d. Crucially, the `ignore` rule *excepts* that theme package from being skipped, while ignoring everything else under `node_modules`. `react`, `react-dom/server`, and `styled-components` are required as side-effect imports to satisfy the theme's peer deps — if they're missing, this throws a hint pointing at `--include=dev`.
2. **Render** — calls `theme.render(resume)` to get full HTML (a `render` export, not the React component).
3. **Font fix-ups** — `mapMissingFonts` rewrites references to `lmsans10-*.otf` (which the theme references but don't exist) to `lmroman10-*.otf` (which are in `fonts/`). `rewriteFontUrls` normalizes every `url(...fonts/*.otf)` reference to the `src/`-relative `../fonts/...` form.
4. **`injectBaseFontSize`** — injects `<style>html{font-size:11.5px}</style>` into `<head>` to scale the theme's `rem`-based layout down.
5. **`injectLinks`** — the brittle part. The theme does not emit `<a>` tags, so this function uses Cheerio to wrap rendered text nodes in links. It matches entries from `resume.json` by normalizing text and finding a matching node among a hardcoded list of **styled-components class selectors** in `titleSelectors` (e.g. `div.sc-hjsuWn.jINFql`). These `sc-*` class names are generated by styled-components and can change if the theme version changes — if a link stops appearing, this is where to debug (add the new selector).

### Data model

`src/resume.json` follows the standard [JSON Resume](https://jsonresume.org/) schema (see `src/schema.json` for the reference shape). The **non-standard extension** here is the optional `website` field on entries — the generator looks for `website` (not the standard `url`) to decide whether to inject a link. Linkable sections and the field used as the link's anchor text:
- `work` → `position`
- `projects` → `name`
- `education` → `institution`
- `awards` → `title`
- `certificates` (or `certifications`) → tries both `title` and `name`
- `references` → `name`

### Deployment (GitHub Pages)

`.github/workflows/pages.yml` runs on every push to `master`: installs with `--include=dev`, runs `generate`, then builds a `site/` artifact. The deploy step rewrites `../fonts/` → `./fonts/` (via `sed`) because the published artifact serves `resume.html` as root `index.html`, not under `src/`. The `fonts/` directory is copied in so paths resolve.

`index.html` at repo root is only for local/preview redirection — it points at `./src/resume.html` and is not part of the deployed artifact.

## Gotchas

- **Dev deps are mandatory**, not optional. The theme is a React app transpiled on the fly. Always install with `--include=dev`.
- **`website`, not `url`**, is the field that triggers link injection. Standard JSON Resume uses `url` for some sections; here that field is ignored by the linker.
- **Styled-components class names are not stable.** If you bump `jsonresume-theme-professional`, the `sc-*` selectors in `injectLinks` may no longer match — links will silently disappear. Verify by generating and inspecting `src/resume.html`.
- `resume.json` lives in `src/`, not the repo root, despite `schema.json` also being present as a reference (it is not consumed by the build).
